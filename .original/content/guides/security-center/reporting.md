## Compliance Reporting

### SOC2 with Opencontrol
See: [Opencontrol Schemas](https://github.com/opencontrol/schemas)

The `data/opencontrols` folder contains the opencontrols data for the ISMS. This is used to generate the compliance report. The report is generated by running `./scripts/compliance_check_soc2.sh` from within the root folder of the repo. This will generate a simple report to stdout. The report is simple and does not contain any of the actual data, only the status of the controls' pesense within the ISMS documentation.

### Last Review Dates

The last review date for each document is stored in the frontmatter of the markdown file. This is used to generate the compliance report. The report is generated by running `./scripts/review_date_check.sh` from within the root folder of the repo. This will generate a simple report to stdout. The report is simple and does not contain any of the actual data, only the status of the last review date for each document.

If desired, the script can be run with an argument to check against a future date. This is useful for checking if a document is due for review.

```bash
# Example check of all documents against today's date
./scripts/review_date_check.sh

# Example check of all documents against a future date
./scripts/review_date_check.sh 2025-01-01
```
